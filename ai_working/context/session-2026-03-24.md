### 3. FR-8.5: Composite Sector View (full feature)
- **Pencil mockups**: 2 screens designed before code:
  - Screen A: Enhanced Sectors page with dashed-border chip selector bar (zone-coloured pips, removable chips, "Analyse N Sectors →" CTA)
  - Screen B: Composite Detail page with blended metrics, unified occupation table with multi-sector badges (HC/PS/Info), dark narrative summary panel
- **Backend**: `GET /api/v1/sectors/composite?codes=62,54,51` — new endpoint in `composite_sector.py`
  - Employment-weighted aggregation across multiple NAICS sectors
  - De-duplicates occupations by SOC (GROUP BY, SUM headcount)
  - `ARRAY_AGG(DISTINCT naics_title)` for per-occupation sector badges
  - Dominant zone from highest-headcount sector, MAX drift velocity (conservative)
  - Validates min 2 codes (400), unknown codes (404)
  - Router registered BEFORE sectors router for literal-first path matching
- **Frontend**:
  - `SectorChipSelector.tsx` — search dropdown, zone-coloured pip chips, analyse button
  - `CompositeSectorPage.tsx` — `/sectors/composite?codes=...` route with metric cards (E0/E1/E2 + purple composite Beta), unified occupation table with sector badge abbreviations, auto-generated narrative summary
  - Integrated into `SectorsPage.tsx`, route added to `App.tsx`
  - API types: `CompositeOccupation`, `CompositeSectorResponse`, `api.compositeAnalysis()`
- **Tests**: 8 backend (TestCompositeSector), 11 component (SectorChipSelector), 10 E2E (composite.spec.ts)
- **Bug fix**: Improved error message when backend not restarted (`2a9699e`)
