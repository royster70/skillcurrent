# Session Summary — 2026-03-26

## What was accomplished

### 1. FR-8.5: Company → Industry Lookup (ASX + LLM Hybrid)

Full feature implementation adding company lookup to the Tier 1 dashboard.

#### ASX company ingestion
- Ingested 1,978 ASX-listed companies with GICS→ANZSIC→NAICS concordance chain
- Hardcoded concordance dict mapping 24 GICS Industry Groups to ANZSIC division letters
- Reverse NAICS lookup via existing `industry_crosswalk` table (FR-8.9 infrastructure)
- pg_trgm fuzzy search index on `company_name` for instant lookup

#### Company search API
- `GET /api/v1/companies/search?q=...&region=AU` — pg_trgm fuzzy search against `asx_company_sectors`
- Returns matching companies with ASX codes, ANZSIC codes, and NAICS codes
- Sub-100ms response times for listed companies

#### LLM classification API
- `POST /api/v1/companies/classify` — Claude Haiku (claude-haiku-4-5-20251001) structured JSON classification
- Prompt includes full sector descriptions (19 ANZSIC divisions for AU, 20 NAICS sectors for US)
- Returns 1-3 sector codes with confidence scores
- Cached in `company_classifications` table — each company classified once

#### CompanyLookup.tsx
- Collapsible purple card on SectorsPage, between ZoneExplainer and SectorChipSelector
- Debounced search input (300ms), ASX code badges on results
- "Classify with AI" button for unlisted companies
- Collapsed by default — zero visual overhead when unused

#### Single-sector smart navigation
- 1 sector → navigate to `/sectors/{code}` (sector detail page)
- 2+ sectors → navigate to `/sectors/composite?codes=...` (composite view)
- Fixed dead-end where single-sector companies were hitting composite view's 2-code minimum validation

## Key architectural decisions

See ADR-005 for full details:
1. ASX + LLM hybrid strategy (instant lookup + long tail classification)
2. GICS → ANZSIC → NAICS concordance chain (reuses FR-8.9 crosswalk)
3. LLM classification with structured output (Claude Haiku, cached)
4. Single-sector navigation path (detail page, not composite)
5. Company lookup as collapsible card (not modal/page)

## Known issues / next steps

- **ANZSIC division-level classification** — too coarse for diversified AU companies. Exploring sub-division ANZSIC classes (506 available) but requires ABS TableBuilder registration for employment data at that level.
- **ANTHROPIC_API_KEY must be in src/backend/.env** (not project root) — Pydantic Settings loads from CWD. Graceful degradation: ASX search works without it, only "Classify with AI" button requires it.
- **Windows uvicorn --reload** — doesn't detect new files; restart needed after adding new endpoint modules (ongoing from previous sessions).
- **US company lookup** — relies entirely on LLM classification (no equivalent of ASX data for US). SEC/EDGAR or D&B integration would improve US coverage.
- **company_classifications cache** — no TTL or invalidation implemented. Stale after mergers/acquisitions until row is manually deleted.
- **Tier 2** features (FR-1 Org Hierarchy → FR-7 Privacy → FR-6 Dashboards) are next in the build chain per CLAUDE.md.
