The key things this gives you:

**Separation of concerns** — data lives at `C:\Projects\data\`, code lives at `C:\Projects\workforce-ai-platform\`. Claude Code knows the path, can reference files in ingestion scripts, but the data isn't cluttering or bloating the repo.

**Version tracking without the data** — when O*NET bumps to 28.2 or a new AEI snapshot drops, you update one line in the registry. Claude Code picks up the new path automatically when building ingestion scripts.

**ADR support** — when Claude Code writes an ADR explaining why Beta = E1 + 0.5×E2, it can cite the exact paper URL and note which local file contains the pre-computed scores it's implementing against.

**Research notes alongside** — you'd add a `docs/research/` folder with short methodology notes per source (the implementable facts extracted from each paper), with the registry pointing to both the notes and the local data path.
